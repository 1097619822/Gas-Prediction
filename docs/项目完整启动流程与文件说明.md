# 项目完整启动流程与文件说明

> 项目名称：基于油田生产数据的天然气产量预测与积液诊断系统  
> 主项目目录：`D:\毕业设计\Project\src_v2`  
> 文档用途：说明项目如何启动、各脚本如何运行、模型文件如何调用，以及主要文件的职责。

---

## 1. 项目总体说明

本项目面向油气井生产数据，完成从 Excel 数据读取、数据清洗、特征工程、天然气产量预测、积液状态诊断，到可视化 Web 演示的完整流程。

核心功能包括：

1. 读取 `data/` 目录下的 30 口井 Excel 生产数据。
2. 对井口压力、套压、日产气量、生产时长等字段做统一抽取。
3. 使用 3-Sigma 和线性插值完成异常值与缺失值处理。
4. 构建滞后特征和滚动窗口特征。
5. 使用 Bi-LSTM 模型进行天然气产量预测。
6. 使用专家规则生成积液诊断标签，并训练积液诊断分类模型。
7. 通过 Streamlit Web 大屏展示井数据、产量预测曲线和积液状态。
8. 生成论文图表和全井汇总表。

## 4. 推荐运行顺序

首次完整运行建议按以下顺序：

1. 安装依赖。
2. 检查 `data/` 目录下 Excel 文件是否存在。
3. 运行 `python -u main_ultra.py` 训练并保存产量预测模型。
4. 运行 `python -u main_diagnosis.py` 训练并保存积液诊断模型。

7. 运行 `python -m streamlit run visualization\web_app.py` 启动 Web 大屏。

如果只是演示系统，并且 `processed_results/` 已经有模型文件，可以直接执行第 7 步。

## 6. 根目录脚本说明

### `config.py`

全局配置文件，定义：

- `BASE_DIR`：当前 `src_v2` 根目录。
- `DATA_DIR`：数据目录，即 `src_v2/data`。
- `OUTPUT_DIR`：结果目录，即 `src_v2/processed_results`。
- `SIGMA_THRESHOLD`：异常值处理阈值，默认 3.0。
- `INTERPOLATE_METHOD`：缺失值插值方式，默认 `linear`。
- `LAG_DAYS`：滞后特征窗口，默认 `[1, 3, 7]`。
- `ROLLING_WINDOWS`：滚动统计窗口，默认 `[3, 7]`。

### `main_ultra.py`

产量预测核心训练脚本。

主要流程：

1. 从 `data/` 中读取第一口井数据，当前默认是 `54-16X.xlsx`。
2. 清洗 `wellhead_press` 和 `gas_volume`。
3. 构建特征：
   - `wellhead_press`
   - `gas_volume_lag_1`
   - `wellhead_press_roll_7_mean`
4. 使用 `StandardScaler` 做标准化。
5. 构建 30 步时间窗口。
6. 训练双向 LSTM 模型。
7. 输出 RMSE、MAE、MAPE、R2。
8. 保存模型权重、完整 checkpoint 和 scaler。

主要输出：

```text
processed_results\ultra_lstm_v2.pth
processed_results\ultra_lstm_v2_checkpoint.pth
processed_results\ultra_lstm_v2_scaler_x.pkl
processed_results\ultra_lstm_v2_scaler_y.pkl
```

### `main_diagnosis.py`

积液诊断模型训练脚本。

主要流程：

1. 遍历 `data/` 下所有井 Excel 文件。
2. 调用专家规则 `DiagnosisRules.apply_rules()` 生成 `diag_label` 和 `diag_state`。
3. 使用 `wellhead_press` 和 `gas_volume` 作为诊断特征。
4. 构建 7 步时间窗口。
5. 汇总所有井的诊断序列样本。
6. 训练 `LSTMClassifier`。
7. 保存诊断模型权重。

主要输出：

```text
processed_results\final_diagnosis_model.pth
```

可通过环境变量控制训练轮数：

```powershell
$env:DIAG_EPOCHS='2'
python -u main_diagnosis.py
```

### `main_with_plots.py`

产量预测训练与论文图表生成组合脚本。它包含与 `main_ultra.py` 类似的训练流程，同时调用 `visualization/thesis_plots.py` 生成多类论文图表。

适合需要“一边训练、一边生成图表”时使用。

### `generate_thesis_figures.py`

论文图表生成入口。它会准备样本数据、训练简化模型，并调用论文图表模块生成：

- 特征相关性热力图
- LSTM 结构图
- 数据预处理流程图
- 预测结果对比图
- 残差分析图
- 模型指标对比图
- 训练曲线图
- 综合评估仪表盘
- 消融实验对比图

### `export_summary.py`

全井汇总导出脚本。它会遍历所有井数据，应用积液诊断规则，并统计每口井的记录数、正常生产天数、积液天数、平均产气量、平均油压等。

输出：

```text
processed_results\全井生产汇总分析表.xlsx
```

## 7. `pre_processing/` 文件说明

### `pre_processing/data_loader.py`

数据加载器。当前主要通过 Excel 的物理列位置抽取字段，而不是完全依赖中文列名。

输出标准字段：

```text
date
wellhead_press
casing_press
gas_volume
prod_hours
```

关键方法：

- `DataLoader.load_well_data(well_filename)`：加载单口井 Excel。
- `DataLoader.get_all_well_files()`：获取 `data/` 下所有 `.xlsx` 文件。

### `pre_processing/cleaner.py`

数据清洗模块。

关键方法：

- `DataCleaner.unify_formats(df)`：统一日期格式。
- `DataCleaner.remove_outliers_3sigma(df, columns)`：使用 3-Sigma 将异常值置为空。
- `DataCleaner.handle_missing_values(df, columns)`：对缺失值做插值和均值兜底。

### `pre_processing/feature_eng.py`

特征工程模块。

关键方法：

- `FeatureEngineer.add_lagged_features()`：添加滞后特征，如 `gas_volume_lag_1`。
- `FeatureEngineer.add_rolling_features()`：添加滚动均值和标准差，如 `wellhead_press_roll_7_mean`。
- `FeatureEngineer.encode_categories()`：类别编码。
- `FeatureEngineer.apply_normalization()`：Min-Max 归一化。

### `pre_processing/diagnosis_rules.py`

专家规则诊断模块。

规则依据：

- 产气量 `gas_volume`
- 井口压力 `wellhead_press`
- 套压 `casing_press`
- 生产时间 `prod_hours`
- 简化 Turner 临界流量曲线

输出标签：

```text
0: Normal
1: WorkWithWater
2: Accumulation
3: Closed
-1: Error
```

## 8. `models/` 文件说明

### `models/lstm_model.py`

产量预测 LSTM 模型。

主要类：

- `LSTMPredictor`：PyTorch LSTM 回归模型，支持多层、双向、Dropout。
- `LSTMTrainer`：训练、验证、预测、保存、加载模型的封装。

### `models/classifier.py`

积液诊断分类模型。

主要类：

- `LSTMClassifier`：双向 LSTM 分类模型。
- `ClassificationTrainer`：分类训练器，使用交叉熵损失。

### `models/baselines.py`

传统基准模型封装。

包括：

- `SVR`
- `RandomForestRegressor`
- `ARIMA`

### `models/informer/`

Informer 长序列预测参考实现目录。

当前目录包含：

```text
main_informer.py
utils/masking.py
utils/metrics.py
utils/timefeatures.py
utils/tools.py
```

注意：当前 `src_v2/models/informer` 只迁移了部分 Informer 文件，缺少原始工程中的部分依赖模块，因此它更适合作为论文对比和后续扩展参考，不建议直接作为当前主流程入口运行。

统一计算回归模型指标。

## 14. 常见问题

### 14.1 Web 提示模型文件不存在

先运行：

```powershell
python -u main_ultra.py
```

确认以下文件存在：

```text
processed_results\ultra_lstm_v2_checkpoint.pth
processed_results\ultra_lstm_v2_scaler_x.pkl
processed_results\ultra_lstm_v2_scaler_y.pkl
```

### 14.2 MAPE 指标非常大

原因通常是真实产量接近 0，百分比误差被放大。建议论文中主要报告：

```text
RMSE
MAE
R2
```

如果必须使用 MAPE，建议过滤低产量点或使用 SMAPE。

### 14.3 `main_diagnosis.py` 运行时间较长

默认训练 30 epoch。如果只是验证流程，可以运行：

```powershell
$env:DIAG_EPOCHS='2'
python -u main_diagnosis.py
```

正式实验再使用默认训练。

### 14.4 中文字体警告

绘图时可能出现 `Glyph missing` 警告，通常是本机缺少中文字体导致，不影响模型训练和数值结果。

### 14.5 Informer 入口不能直接运行

`models/informer/` 当前只迁移了部分 Informer 文件，缺少原始工程的部分依赖。当前主流程建议使用 `main_ultra.py` 的 Bi-LSTM 模型；Informer 可作为论文对比和后续扩展方向。

## 16. 当前项目主线结论

当前项目可按如下主线理解：

```text
Excel 原始数据
  -> DataLoader 标准字段抽取
  -> DataCleaner 异常值和缺失值处理
  -> FeatureEngineer 时序特征构建
  -> LSTMPredictor 产量预测
  -> DiagnosisRules / LSTMClassifier 积液诊断
  -> Evaluator 指标评估
  -> WebApp / ThesisPlots 可视化输出
```

正式演示时建议优先使用完整 checkpoint：

```text
processed_results\ultra_lstm_v2_checkpoint.pth
```

它比单独的 `.pth` 权重更适合后续直接调用和复现实验结果。
